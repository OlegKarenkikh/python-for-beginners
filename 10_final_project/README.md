## Что делает скрипт

1. Читает базу клиентов из `clients.json`
2. Для каждого клиента рассчитывает страховую премию
3. Проверяет заявления на соответствие правилам
4. Сохраняет результаты в `results.json`
5. Выводит сводную статистику

## 🔍 Урок на внимательность: граничные условия

Функция `age_factor()` использует **граничное условие** `age <= 35`, чтобы включить
клиентов ровно 35 лет в группу «25–35 лет» (коэффициент 1.1).

```python
# ✅ Правильно: <= включает граничное значение (35 лет входит в группу 25–35)
if age <= 35: return 1.1

# ❌ Неправильно: < исключает 35 (клиент 35 лет попадает в следующую группу!)
if age < 35:  return 1.1
```

> **Тест как страховка**: именно `test_age_factor_normal()` поймал бы эту ошибку.
> Запускайте `pytest tests/` после каждого изменения в коде — это профессиональная привычка.

### 🙋 Финальный проект «сделан» — как понять что хорошо?

Три признака хорошего финального проекта:
1. **Запускается без ошибок**: `python premium_calculator.py` проходит от начала до конца
2. **Тесты проходят**: `pytest tests/` — все зелёные
3. **Читаемый код**: другой человек понимает что делает каждая функция

```bash
cd 10_final_project
python premium_calculator.py   # должен распечатать статистику
pytest tests/ -v               # должны пройти все тесты
```
