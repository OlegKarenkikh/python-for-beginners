## 🔌 Глава 13: Реестр функций (Plug & Socket)

**Где болит:** Код `TOOL_REGISTRY[name](**args)` выглядит непонятным для новичка.

**Аллегория: «Розетка и Вилка»**

Представь, что твои функции — это электроприборы:
1.  `calculate_premium()` — это **Чайник**.
2.  `check_documents()` — это **Лампа**.
3.  `find_client()` — это **Радио**.

Когда ты пишешь `def calculate_premium(...)`, ты просто собираешь Чайник на заводе. Чтобы он заработал, его нужно включить в розетку.

`TOOL_REGISTRY` — это сетевой фильтр (удлинитель) с наклейками над каждой розеткой:
```python
TOOL_REGISTRY = {
    "чайник": calculate_premium,  # Наклейка "чайник" -> Прибор Чайник
    "лампа":  check_documents,   # Наклейка "лампа"   -> Прибор Лампа
}
```

Агент (LLM) — это человек, который не умеет пользоваться приборами, но умеет читать наклейки. Он говорит: «Мне нужно вскипятить воду, включи **чайник** с параметром **температура=100**».

Python делает три шага:
1.  `name = "чайник"` (LLM дала имя).
2.  `func = TOOL_REGISTRY[name]` (Мы нашли прибор по наклейке).
3.  `func(temperature=100)` (Мы включили прибор в сеть с нужной настройкой).

**Код «мостика»:**
```python
# Вместо этого:
# result = TOOL_REGISTRY[tool_name](**tool_args)

# Думай так:
прибор = TOOL_REGISTRY[имя_от_нейросети]  # Найти функцию по имени
результат = прибор(**настройки_от_нейросети)  # Запустить функцию с данными
```

## 🧪 Глава 11: FastAPI и Валидация (Воронка с ситом)

**Где болит:** Зачем нужны `BaseModel` и `Field`, если можно просто принимать `dict`?

**Аллегория: «Воронка с ситом»**

Представь, что твой API — это вход в банк.
*   **Обычный `dict`** — это если бы все посетители просто закидывали кучу документов в одну коробку. Тебе пришлось бы самому разбираться: где паспорт, где сумма, не подделка ли это?
*   **Pydantic (`BaseModel`)** — это автоматический турникет с датчиками.

Когда данные (JSON) попадают в `ClientRequest(BaseModel)`:
1.  **Сито типов:** Если вместо возраста (число) прислали слово «тридцать», турникет не пропустит человека (ошибка 422).
2.  **Датчик `Field(ge=18)`:** Если человеку 15 лет, турникет сработает и скажет: «Рано еще страховаться!» (ошибка 422).

**Преимущество:** Твой код внутри функции (логика расчёта) теперь в полной безопасности. Он знает, что если данные прошли через «турникет» Pydantic, они 100% верные и чистые.

## 📦 Глава 09: Модули (Библиотека vs Инструмент)

**Где болит:** Когда писать код в `main.py`, а когда создавать папку и `__init__.py`?

**Аллегория: «Мастерская и Склад»**

1.  **Скрипт (`main.py`)** — это твой рабочий верстак. Здесь ты собираешь конечный продукт (запускаешь программу). Здесь лежит молоток и пила.
2.  **Модуль (`utils.py`)** — это ящик с инструментами. Ты не живешь в ящике, ты достаешь из него отвертку, когда она нужна.
3.  **Пакет (Папка с `__init__.py`)** — это целый складской отдел (например, «Отдел сантехники»).

**Правило:**
*   Если код «делает дело» (считает, выводит в консоль) — это **Скрипт**.
*   Если код «лежит и ждет» (просто описание функций) — это **Модуль**.
*   Никогда не делай `import` из Скрипта в Модуль. Инструменты не должны зависеть от верстака!
