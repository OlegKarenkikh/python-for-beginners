## Другая аналогия: ячейки в памяти компьютера

Если ящики со шкафом не зашли — попробуйте этот взгляд:

Компьютер — это очень большой блокнот с миллиардами ячеек.
Каждой ячейке можно дать имя. Python делает это за вас автоматически.

```python
# Вы пишете:
age = 35

# Python делает:
# 1. Находит свободную ячейку в памяти (например, адрес #7F3A20)
# 2. Записывает туда число 35
# 3. Запоминает: "age" → #7F3A20
# 4. Когда вы пишете age — Python ищет по имени и возвращает 35
```

**Главная ловушка:**
```python
age = 35
age = 22   # старое значение 35 ИСЧЕЗАЕТ навсегда!
print(age) # → 22
```

Переменная — это не математическая переменная x.
Это скорее «последнее сохранённое значение».

## Страшная сторона: типы нельзя смешивать

```python
age = 35
name = "Иванов"

# Это вызовет ошибку!
result = age + name
# TypeError: unsupported operand type(s) for +: 'int' and 'str'
```

**Почему?** Python не знает как сложить число и слово.
35 + "Иванов" — что должно получиться? "35Иванов"? 35? Ошибка.

```python
# Исправление 1: преобразование
result = str(age) + name       # "35Иванов" — строки склеились
result = "Возраст: " + str(age)  # "Возраст: 35"

# Исправление 2: f-строка (лучший способ!)
result = f"Клиент {name}, возраст {age}"  # "Клиент Иванов, возраст 35"
```

## Страшная сторона: None — это не ноль

```python
# Пользователь ещё не ввёл возраст
age = None

# Опасно! None + 5 → ошибка
premium = None

if premium is None:
    print("Премия ещё не рассчитана")
else:
    print(f"Премия: {premium} руб.")
```

`None` — это «ничего», «пусто», «не установлено».
Не путайте с `0` (ноль) или `""` (пустая строка) — это разные вещи.

| Значение | Смысл |
|---|---|
| `None` | Неизвестно / не установлено |
| `0` | Известно — равно нулю |
| `""` | Известно — пустая строка |
| `False` | Известно — ложь |

## Страшная сторона: тихие ошибки с числами

```python
# Выглядит правильно, но результат неожиданный
age = "35"   # СТРОКА, не число!
age = age + 5
# TypeError: can only concatenate str (not "int") to str

# Если данные пришли из формы или файла — они всегда строки!
age_from_form = "35"    # строка
age = int(age_from_form)  # явно преобразуем
age = age + 5           # теперь 40 ✓
```

## Правило: всегда знайте тип

```python
# Инструмент отладки
age = "35"
print(type(age))      # <class 'str'> — строка!

age = int(age)
print(type(age))      # <class 'int'> — теперь число

premium = 12_000
print(type(premium))  # <class 'int'>

coeff = 1.5
print(type(coeff))    # <class 'float'>
```

### 🙋 Почему кавычки обязательны для текста?

Кавычки сообщают Python: «это текст, не команда».
Без кавычек Python подумает что `Иванов` — это имя переменной и скажет `NameError`.

```python
name = Иванов     # ❌ NameError: name 'Иванов' is not defined
name = "Иванов"   # ✅ OK — это текстовая строка
```

---

### 🙋 Почему `True` с заглавной буквы? Что если написать `true`?

Python чувствителен к регистру. `True` — встроенное ключевое слово.
`true` Python воспримет как имя переменной и скажет `NameError`.

```python
flag = True    # ✅ правильно
flag = true    # ❌ NameError: name 'true' is not defined
flag = False   # ✅ правильно
flag = None    # ✅ правильно (тоже с заглавной)
```

**Запомните:** `True`, `False`, `None` — всегда с заглавной буквы.

### 🙋 Почему точка в числах, а не запятая?

Python использует международный стандарт: десятичный разделитель — точка.
Запятая в Python имеет другой смысл (разделяет элементы в списке или кортеже).

```python
premium = 45000.50    # ✅ дробное число
premium = 45000,50    # ❌ это кортеж из двух чисел: (45000, 50)!
```

Чтобы визуально разбить большое число на группы — используйте `_`:

```python
big_sum = 1_500_000   # ✅ читабельно, Python игнорирует подчёркивания
big_sum = 1500000     # тоже ✅ но труднее читать
```

### 🙋 Что значит `:.0f` внутри f-строки?

Это спецификация формата числа. Расшифровка:
- `:` — начало формата
- `.0` — ноль знаков после запятой
- `f` — дробное число (float)

```python
x = 18000.5678
print(f"{x:.0f}")    # 18001      — ноль знаков после запятой
print(f"{x:.2f}")    # 18000.57   — два знака
print(f"{x:,.0f}")   # 18,001     — с разделителем тысяч
```

### 🙋 Старое значение переменной исчезает навсегда?

В большинстве случаев — да. Если нужно сохранить — запишите в другую переменную:

```python
old_age = age   # сохранили старое значение
age = 22        # изменили
# old_age = 35, age = 22 — оба доступны
```

### 🙋 Почему `is None`, а не `== None`?

`is` проверяет тождественность (один и тот же объект в памяти).
`None` в Python всегда один экземпляр, поэтому правильно писать `is None`.

```python
age = None

if age is None:       # ✅ правильный питоновский стиль
    print("не указан")

if age == None:       # ⚠️ работает, но линтер будет ругаться
    print("не указан")
```

> 💡 Выработайте рефлекс: `is None` и `is not None` — всегда так.
