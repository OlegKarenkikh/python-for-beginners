## Раздел 00 — До старта

**Q1. Что значит `../`?**

`../` — это «выйди из текущей папки на уровень вверх».
Представьте файловую систему как дерево папок:
```
python-for-beginners/
├── 00_for_beginners/   ← вы здесь
│   └── README.md
└── 00_foundations/
    └── 01_what_is_python.md
```
Из `00_for_beginners/` путь `../00_foundations/01_what_is_python.md` означает:
«поднимись наверх (../) и зайди в 00_foundations».
Одна точка `.` = текущая папка. Две точки `..` = папка выше.

**Q2. Что такое терминал?**

Терминал — это программа где вы набираете команды текстом, а не кликаете мышкой.

- Windows: нажмите Win+R, наберите `cmd` или `powershell`
- Mac: Spotlight (Cmd+Space), наберите `Terminal`
- VS Code: меню Terminal → New Terminal

Терминал, консоль, командная строка — в быту это синонимы.

**Q3. Нужно знать страхование?**

Нет. Страхование — просто контекст. Все примеры понятны и без него:
«клиент» = любой человек, «премия» = сумма денег, «заявление» = запрос.
Это мог быть любой бизнес: магазин, банк, больница.

## Глава 01 — Переменные

**Q4. Зачем кавычки для текста?**

Кавычки сообщают Python: «это текст, не команда».
Без кавычек Python подумает что `Иванов` — это имя переменной или функции.
```python
name = Иванов     # NameError: name 'Иванов' is not defined
name = "Иванов"   # OK — это текстовая строка
```
Кавычки обязательны для любого текстового значения.

**Q5. Почему `True` с большой буквы? Что если `true`?**

Python чувствителен к регистру. `True` — это встроенное ключевое слово языка.
`true` (маленькими) — Python посчитает это именем переменной и скажет `NameError`.
Запомните раз и навсегда: `True`, `False`, `None` — всегда с заглавной буквы.

**Q6. Почему точка, а не запятая в числах?**

Python использует международный стандарт где десятичный разделитель — точка.
Запятая в Python имеет другой смысл (разделяет элементы).
`45000,50` Python прочитает как два числа: `45000` и `50`.
Чтобы визуально разделить тысячи — используйте `_`: `45_000.50` — корректно!

**Q7. Что такое `f` перед строкой и фигурные скобки?**

`f"..."` — это f-строка (formatted string). `f` говорит Python:
«внутри строки есть выражения в `{}` — вычисли их и подставь результат».

```python
name = "Иванов"
age = 35
print(f"Клиент: {name}, возраст: {age}")
# Python вычисляет {name} → "Иванов", {age} → 35
# Результат: "Клиент: Иванов, возраст: 35"
```

Без `f` фигурные скобки напечатаются буквально: `"Клиент: {name}"`.

**Q8. Что такое `:.0f`?**

Это формат числа внутри f-строки. Расшифровка:
- `:` — начало спецификации формата
- `.0` — ноль знаков после запятой
- `f` — float (дробное число)

Примеры:
```python
x = 18000.5678
print(f"{x:.0f}")    # 18001   — ноль знаков
print(f"{x:.2f}")    # 18000.57 — два знака
print(f"{x:,.0f}")   # 18,001  — запятая-разделитель тысяч
print(f"{x:.1%}")    # 1800056.8% — как процент (умножает на 100)
```

**Q9. Что будет если написать `2name = "Иванов"`?**

Немедленная ошибка синтаксиса ещё до запуска:
```
SyntaxError: invalid decimal literal
```
Python видит цифру и думает что вы пишете число, потом встречает букву — не понимает.
Программа даже не начнёт выполняться.

**Q10. Что такое `<class 'int'>`?**

`class` здесь — техническое слово Python для описания «типа» данных.
Не пугайтесь: `<class 'int'>` просто означает «целое число».
Классы вы изучите позже. Сейчас достаточно знать:
`type(x)` показывает что хранится в переменной: строка, число, список и т.д.

## Глава 01 DEEP_DIVE

**Q11. В других языках память выделяют вручную?**

Да. В C/C++ вы пишете `malloc(sizeof(int))` чтобы выделить память и `free()` чтобы освободить.
Забыли освободить — утечка памяти. Python делает это автоматически через «сборщик мусора».
Это одна из причин почему Python легче для начала.

**Q12. Можно ли увидеть реальный адрес памяти?**

Да! Функция `id()`:
```python
age = 35
print(id(age))   # например 140234567890192
```
Это реальный адрес в памяти. Но в повседневной работе он не нужен.
Полезно только при отладке сложных проблем с изменяемыми объектами.

**Q13. Старое значение исчезает навсегда?**

В большинстве случаев — да. Но Python хранит небольшие целые числа (-5 до 256) в кэше.
На практике: **считайте что исчезает**. Не рассчитывайте на кэш — это деталь реализации.
Если нужно сохранить старое значение — запишите в другую переменную:
```python
old_age = age
age = 22
# old_age = 35, age = 22
```

**Q14. Что такое `db.query()`?**

`db` — это объект подключения к базе данных (например PostgreSQL или SQLite).
В реальном коде вы создаёте соединение: `db = connect("postgresql://...")`.
Потом вызываете `.query()`. В курсе это пример из реального мира — не запускаемый пример.

**Q15. Почему `is None`, а не `== None`?**

`is` проверяет тождественность (один и тот же объект в памяти).
`==` проверяет равенство значений.

`None` в Python всегда один — это синглтон. Поэтому правильно: `if x is None`.
`x == None` тоже работает, но Python Linter (ruff/flake8) будет ругаться и предложит заменить.
Привыкайте писать `is None` — это идиома Python.

**Q16. Как долго `input()` ждёт ввода?**

Вечно, пока пользователь не нажмёт Enter. Программа «замирает» на этой строке.
В реальных веб-приложениях `input()` не используется — там данные приходят через HTTP-запросы.
`input()` полезен только для скриптов в терминале.

## Глава 02 — Строки

**Q17. Что если внутри строки нужна двойная кавычка?**

Три способа:
```python
# 1. Использовать одинарные снаружи
text = 'Клиент сказал "Одобрено"'

# 2. Экранировать обратным слешем
text = "Клиент сказал \"Одобрено\""

# 3. Тройные кавычки
text = """Клиент сказал "Одобрено" и подписал"""
```
Я предпочитаю способ 1 или 3 — они читаемее.

**Q18. Что если несколько пробелов подряд при `split()`?**

`split()` без аргументов — умный: игнорирует любое количество пробелов и переносов строк.
```python
"  Иванов   Алексей  ".split()   # ['Иванов', 'Алексей']
```
Но `split(" ")` (с явным пробелом) — тупой: оставляет пустые строки:
```python
"  Иванов   Алексей  ".split(" ")   # ['', '', 'Иванов', '', '', 'Алексей', '', '']
```
**Используйте `split()` без аргументов** для текста с пробелами.

**Q19. Можно ли цепочкой: `name.split()[0]`?**

Да, и это очень распространённый паттерн. Называется «method chaining» (цепочка методов).
```python
name = "  иванов алексей петрович  "
first = name.strip().title().split()[0]
# strip()  → "иванов алексей петрович"
# title()  → "Иванов Алексей Петрович"
# split()  → ["Иванов", "Алексей", "Петрович"]
# [0]      → "Иванов"
```
Читайте слева направо: каждый шаг берёт результат предыдущего.

**Q20. Почему `name.upper()` не сохраняется?**

Потому что строки **неизменяемы**. `upper()` создаёт новую строку, но никуда её не кладёт.
Это как спросить официанта «принесите счёт» — он принёс и положил на стол.
Если вы не взяли — он унесёт. Сохраняйте явно:
```python
name = name.upper()   # перезаписать
upper_name = name.upper()   # в новую переменную
```

## Глава 02 DEEP_DIVE

**Q21. `name = name.strip()` — это изменение строки или новая переменная?**

Новая строка! Правая часть `name.strip()` создаёт новую строку.
Потом `name =` делает так, что имя `name` начинает указывать на новую строку.
Старая строка никуда не делась — просто теперь на неё никто не ссылается, и Python её удалит.

Технически: переменная `name` — это «ярлык». Вы переклеиваете ярлык на новый объект.

**Q22. Что такое кэш строк? Зачем это знать?**

Python оптимизирует память: если две переменные содержат одинаковую строку-литерал, они могут указывать на один объект в памяти. Для вас как новичка это неважно — просто не пишите `if x is "Иванов"` (сравнивайте через `==`). Как только это делаете — нарушаете правила и получаете неожиданное поведение.

**Q23. Что такое `frozenset`?**

`set` — множество изменяемое: можно добавлять/удалять элементы.
`frozenset` — множество «замороженное»: изменить нельзя, зато можно использовать как ключ словаря.

```python
cities = frozenset({"Москва", "СПб", "Казань"})
"Москва" in cities   # True — мгновенная проверка O(1)
```

Используется когда хотите сказать «этот набор не изменится никогда».

**Q24. Зачем кортеж если есть список?**

Семантика. Список — «коллекция однотипных вещей которую можно менять».
Кортеж — «запись с фиксированной структурой».

```python
# Список — однотипные вещи
names = ["Иванов", "Петрова", "Сидоров"]

# Кортеж — разнотипные поля одной записи (имя, возраст, флаг)
client = ("Иванов", 35, True)
```

Плюс кортежи быстрее списков и могут быть ключами словаря.

## Глава 03 — Числа

**Q25. Подчёркивание в числе `1_500_000` — это работает?**

Да, абсолютно корректно начиная с Python 3.6. Это просто косметика для читаемости.
Python игнорирует `_` в числах. `1_500_000 == 1500000` — True.
Удобно для больших чисел: `1_000_000` vs `1000000` — второе легко перепутать с `100_000`.

**Q26. `4 / 2` — это 2.0 или 2?**

В Python 3 — `2.0`. Всегда float. Проверьте:
```python
print(4 / 2)       # 2.0
print(type(4 / 2)) # <class 'float'>
print(4 // 2)      # 2  (целочисленное)
```
Это кардинально изменилось с Python 2 где `4 / 2 = 2`. Python 2 мёртв — забудьте о нём.

**Q27. `from decimal import Decimal` — скачивать надо?**

Нет! `decimal` — стандартная библиотека Python, устанавливается вместе с Python.
`import` — не значит «скачать». Значит «подключить модуль из уже установленного».
Скачивать надо только сторонние пакеты через `pip install`.

**Q28. Зачем `Decimal("0.1")` строкой, не числом?**

Если написать `Decimal(0.1)` — вы передадите уже неточное float значение.
Python сначала создаст `0.1` как float (уже неточный!), потом передаст в Decimal.
`Decimal("0.1")` — строка парсится точно, без промежуточного float.

```python
from decimal import Decimal
print(Decimal(0.1))    # 0.1000000000000000055511151231257827021181583404541015625 (!!)
print(Decimal("0.1"))  # 0.1 — точно
```

**Q29. Почему `0.1 + 0.2 = 0.30000000000000004`?**

Компьютер хранит числа в двоичной системе. Некоторые дроби не представимы точно в двоичном виде — как `1/3` в десятичной (0.333...).
`0.1` в двоичном — бесконечная дробь. Компьютер обрезает её — и появляется ошибка.

Это не баг Python — так работают все современные процессоры (стандарт IEEE 754).
**Решение:** `Decimal` для денег, `round()` для вывода, `math.isclose()` для сравнения.

## Глава 04 — Условия

**Q30. Двоеточие после `if` — обязательно?**

Да, абсолютно обязательно. Двоеточие говорит Python: «дальше начинается блок».
`SyntaxError: expected ':'` — одна из самых частых ошибок новичков.
После `if`, `elif`, `else`, `for`, `while`, `def`, `class` — всегда двоеточие.

**Q31. `risk *= 1.5` — это сокращение?**

Да. Операторы присваивания с операцией:
```python
x *= 2    # x = x * 2
x += 5    # x = x + 5
x -= 3    # x = x - 3
x /= 2    # x = x / 2
x //= 2   # x = x // 2
x %= 3    # x = x % 3
x **= 2   # x = x ** 2
```
Все работают одинаково — сокращённая запись.

**Q32. `not experienced` — отличается от `experienced == False`?**

Функционально часто одинаково, но `not` — универсальнее.
`not x` — истина если `x` «ложное» (False, None, 0, "", [], {}).
`x == False` — истина только если `x` именно False.

```python
x = 0
not x          # True (0 — ложное)
x == False     # True (0 == False в Python — да, это нюанс!)

x = []
not x          # True (пустой список — ложный)
x == False     # False ([] != False)
```

Используйте `not x` — это питоновский стиль.

**Q33. Тернарный оператор — когда использовать?**

Когда значение зависит от одного простого условия и обе ветки — одно выражение.
```python
# Хорошо — коротко и ясно
label = "молодой" if age < 25 else "стандарт"

# Плохо — слишком длинно, лучше обычный if
result = calculate_very_long_function(a, b) if some_complex_condition(x, y) else another_long_function(c, d)
```
Правило: если тернарный оператор не умещается в 79 символов — используйте обычный if.

## Глава 05 — Циклы

**Q34. Переменная в `for name in claims:` — объявляется здесь?**

Да, прямо здесь. Python создаёт переменную `name` при первой итерации.
Важный нюанс: после цикла переменная **остаётся** со значением последней итерации:
```python
for name in ["Иванов", "Петрова"]:
    pass
print(name)   # "Петрова" — переменная жива после цикла!
```
Это иногда удивляет. В реальном коде лучше не полагаться на это — явно объявляйте заранее.

**Q35. `total += p` — сокращение от `total = total + p`?**

Да, полностью эквивалентно. Только убедитесь что `total` уже объявлен до цикла:
```python
total = 0       # ← обязательно! иначе NameError
for p in premiums:
    total += p
```

**Q36. `range(1, 6)` — конец включается?**

Нет! `range(start, stop)` — stop **не включается**.
`range(1, 6)` = 1, 2, 3, 4, 5.
Запомните так: если хотите числа от 1 до N включительно — пишите `range(1, N+1)`.
Это согласуется со срезами: `names[1:3]` — тоже не включает 3.

**Q37. `claims[:]` — зачем так копировать?**

`claims[:]` — срез «от начала до конца» — создаёт поверхностную копию списка.
Это краткая запись `claims[0:len(claims)]` или `list(claims)`.
Используется когда хотите итерировать по оригиналу, но изменять — без риска пропустить элементы.

**Q38. Что за конструкция `next((c for c in ...), None)`?**

`(c for c in ... if ...)` — это генераторное выражение (generator expression).
Как list comprehension `[...]`, но ленивое: вычисляет по одному элементу, а не весь список сразу.
`next(generator, default)` — берёт первый элемент, или возвращает `default` если пусто.
Эффективно: как только нашли — останавливаемся, не перебираем весь список.

**Q39. Что такое `collections`?**

Стандартная библиотека Python. Не требует `pip install`.
Содержит полезные структуры данных:
- `defaultdict` — словарь с дефолтным значением для новых ключей
- `Counter` — считает количество вхождений
- `OrderedDict` — словарь с гарантированным порядком (в Python 3.7+ обычный dict тоже упорядочен)
- `deque` — двусторонняя очередь

## Глава 06 — Функции

**Q40. Параметры без типов — как Python знает что передавать?**

Никак. Python — динамически типизированный язык: типы проверяются только во время выполнения.
Вы можете передать что угодно — Python не остановит. Ошибка будет позже, когда код попытается что-то сделать с неправильным типом.

Для подсказок используют аннотации типов (не обязательные):
```python
def calc(age: int, base: float) -> float:
    ...
```
Это документация для человека и IDE. Python их не проверяет — проверяет `mypy`.

**Q41. Тройные кавычки в начале функции — что это?**

Это **docstring** — документационная строка. Не комментарий! Хранится в `__doc__`:
```python
def calc(age, base):
    """Рассчитывает премию. Возраст в годах, база в рублях."""
    ...

print(calc.__doc__)  # "Рассчитывает премию. Возраст в годах, база в рублях."
help(calc)           # выводит docstring красиво
```
IDE (VS Code, PyCharm) показывает docstring во всплывающей подсказке при наведении.

**Q42. `round()` откуда взялась без import?**

Это встроенная функция (builtin). Python загружает их автоматически.
Полный список буiltins: `print`, `len`, `type`, `int`, `str`, `float`, `bool`, `list`, `dict`, `set`, `tuple`, `range`, `enumerate`, `zip`, `map`, `filter`, `sorted`, `sum`, `min`, `max`, `abs`, `round`, `input`, `open`, `isinstance`, `hasattr` и другие (~70 штук).
Всё что не в этом списке — нужно импортировать.

**Q43. Что значит `{client_id:05d}`?**

- `05` — ширина поля 5 символов, заполнить нулями слева
- `d` — decimal (целое десятичное число)

```python
print(f"{42:05d}")    # 00042
print(f"{1:05d}")     # 00001
print(f"{12345:05d}") # 12345
print(f"{123456:05d}")# 123456 — шире 5, не обрезается!
```

**Q44. При вызове именованных аргументов — порядок важен?**

При именованных — нет:
```python
def f(a, b=10, c=20): ...
f(1, c=30, b=5)   # OK: a=1, b=5, c=30
```
При позиционных — порядок строгий.
Правило: именованные аргументы всегда идут после позиционных.

**Q45. «Функция — объект» — что это значит?**

В Python всё — объект. Функции тоже. Это значит что функцию можно:
```python
def greet(name):
    return f"Привет, {name}!"

# Присвоить переменной
say_hi = greet
say_hi("Иванов")   # "Привет, Иванов!"

# Передать как аргумент
def apply(func, name):
    return func(name)

apply(greet, "Петрова")   # "Привет, Петрова!"

# Положить в список
operations = [greet, str.upper, str.lower]
```
Это мощная концепция — основа декораторов и функционального программирования.

## Глава 07 — Списки и словари

**Q46. Почему `[-1]` последний, а не `[-0]`?**

`-0` == `0` в Python (и вообще в математике). Поэтому `[-0]` — это `[0]` — первый элемент.
Отрицательные индексы — синтаксический сахар для `len(lst) + index`:
- `[-1]` = `len - 1` = последний
- `[-2]` = `len - 2` = предпоследний

**Q47. Почему «конец среза не включается»?**

Дизайнерское решение Гвидо ван Россума, создателя Python. Преимущества:
1. `len(lst[0:n])` == `n` — интуитивно
2. `lst[0:3]` + `lst[3:6]` == `lst[0:6]` — срезы стыкуются без перекрытий
3. `lst[:n]` — первые n элементов (а не n-1)

Большинство программистов через месяц перестают думать об этом.

**Q48. `dict.get()` vs `dict["key"]` — в чём разница?**

```python
client = {"name": "Иванов", "age": 35}

client["city"]        # KeyError — такого ключа нет, ошибка!
client.get("city")    # None — тихо возвращает None
client.get("city", "Не указан")  # "Не указан" — дефолтное значение
```

Используйте `["key"]` когда ключ ОБЯЗАН быть (ошибка — ожидаемая).
Используйте `.get()` когда ключ МОЖЕТ отсутствовать (опциональные поля).

**Q49. Что такое `.items()` и две переменные в for?**

`.items()` возвращает пары (ключ, значение) — список кортежей.
В Python можно «распаковать» кортеж в несколько переменных прямо в `for`:
```python
pair = ("name", "Иванов")
key, value = pair   # распаковка

# В цикле — то же самое:
for key, value in client.items():
    print(key, value)
```
Это называется tuple unpacking (распаковка кортежа).

**Q50. List comprehension — что это?**

Компактный способ создать список из другого списка с трансформацией/фильтрацией:
```python
# Обычный for
result = []
for c in database:
    if c["city"] == "Москва":
        result.append(c)

# List comprehension — то же самое, одной строкой
result = [c for c in database if c["city"] == "Москва"]

# Структура: [что_добавить  for переменная in итерируемое  if условие]
squares = [x**2 for x in range(10) if x % 2 == 0]
```

**Q51. `sum(c["premium"] for c in database)` — без скобок списка?**

Это **генераторное выражение** — как list comprehension, но без создания промежуточного списка.
`sum([x for x in ...])` — создаёт список, потом суммирует.
`sum(x for x in ...)` — суммирует на лету, не создавая список в памяти.
При большом количестве данных — значительная экономия памяти.

## Глава 08 — Файлы и JSON

**Q52. Что такое `as f`?**

`as f` — это псевдоним (alias). Вы говорите: «объект файла называть `f` внутри этого блока».
Имя может быть любым (`as file`, `as fh`, `as fp`), но `f` — стандартное соглашение.
`f` — объект файла с методами `read()`, `write()`, `readline()` и др.

**Q53. Все режимы открытия файла:**

```
"r"  — чтение (по умолчанию). Ошибка если файл не существует.
"w"  — запись. Создаёт файл. ПЕРЕЗАПИСЫВАЕТ если существует!
"a"  — добавление (append). Добавляет в конец, не стирает.
"x"  — создать новый. Ошибка если файл уже существует.
"rb" — чтение в бинарном режиме (для изображений, PDF)
"wb" — запись в бинарном режиме
```
Самая опасная ловушка: `"w"` молча сотрёт файл если он существует.

**Q54. `json.dump` vs `json.dumps`:**

```
json.dump(obj, file)   — записывает в файл (d = destination?)
json.dumps(obj)        — возвращает строку (s = string)

json.load(file)        — читает из файла
json.loads(text)       — парсит строку
```
Мнемоника: `s` в конце = string (работа со строками, не файлами).

**Q55. `ensure_ascii=False` — что это?**

По умолчанию `json.dumps` экранирует все не-ASCII символы:
```python
json.dumps({"name": "Иванов"})
# '{"name": "\\u0418\\u0432\\u0430\\u043d\\u043e\\u0432"}'  ← нечитабельно!

json.dumps({"name": "Иванов"}, ensure_ascii=False)
# '{"name": "Иванов"}'  ← читабельно
```
Всегда используйте `ensure_ascii=False` при работе с кириллицей.

**Q56. Python конвертирует `True`/`False` автоматически?**

Да, `json.dump`/`json.load` делают это за вас:
```python
Python → JSON:  True → true,  False → false,  None → null
JSON → Python:  true → True,  false → False,  null → None
```

## Глава 11 — FastAPI

**Q57. Что такое `@app.get("/")`?**

`@` — это **декоратор**. Он модифицирует функцию после неё.
`@app.get("/")` означает: «зарегистрируй следующую функцию как обработчик GET-запроса на путь `/`».

Технически `@decorator` — это сокращение для:
```python
def home(): ...
home = app.get("/")(home)
```
Декораторы — мощная концепция, используется везде: FastAPI, тесты, кэширование.

**Q58. Откуда FastAPI знает что вызывать?**

FastAPI читает декоратор `@app.get("/")` и сохраняет функцию как обработчик этого маршрута.
Когда приходит GET-запрос на `/` — FastAPI находит зарегистрированную функцию и вызывает её.
Аргументы FastAPI достаёт автоматически из URL, тела запроса или заголовков — по именам параметров функции.

**Q59. `uvicorn main:app` — имя файла и переменная?**

Да точно. `main:app` расшифровывается:
- `main` — файл `main.py` (без расширения)
- `app` — переменная `app = FastAPI()` внутри этого файла

`--reload` — перезапускать при изменении файлов (только для разработки!).

**Q60. `class ClientRequest(BaseModel)` — что за класс?**

Класс — это шаблон для создания объектов с данными и методами.
`BaseModel` из Pydantic — специальный класс для валидации данных.
Когда вы пишете `class ClientRequest(BaseModel)` — создаёте схему:
«ожидаю объект с полями name (str), age (int), accidents (int)».
Pydantic автоматически проверит что данные из запроса соответствуют схеме.

**Q61. `ge=18, le=90` — что это?**

- `ge` — greater or equal (≥ 18)
- `le` — less or equal (≤ 90)
- `gt` — greater than (>)
- `lt` — less than (<)

Pydantic автоматически вернёт HTTP 422 если значение не соответствует.

**Q62. `raise` — что это?**

`raise` — поднять исключение. Намеренно «сломать» выполнение с сообщением об ошибке.
```python
raise HTTPException(status_code=404, detail="Клиент не найден")
```
FastAPI перехватит это исключение и вернёт клиенту JSON с кодом 404.
`raise` используется когда что-то пошло не так и нужно прервать выполнение.

**Q63. Зачем `response_model`?**

1. **Документация** — Swagger UI (`/docs`) покажет правильную схему ответа
2. **Фильтрация** — лишние поля из вашего объекта не попадут в ответ
3. **Валидация** — если вы вернёте данные неправильного типа — ошибка на этапе разработки

## Глава 12 — LLM

**Q64. Что такое `"role"` в messages?**

OpenAI и совместимые API (Ollama, vLLM) ожидают разговор в виде списка сообщений с ролями:
- `"system"` — инструкции для модели («ты — страховой агент»)
- `"user"` — сообщение от пользователя («посчитай премию»)
- `"assistant"` — предыдущий ответ модели (для сохранения контекста разговора)

Это позволяет модели понимать контекст многоходового диалога.

**Q65. `stream=True` — что такое поток?**

Без stream: модель думает 10 секунд, потом выдаёт весь ответ сразу.
С stream: модель выдаёт ответ по словам/кускам по мере генерации.

В интерфейсе это выглядит как «живой» набор текста.
Технически ответ приходит как HTTP Server-Sent Events — поток данных.

**Q66. `for chunk in response` — что такое chunk?**

При `stream=True` response — итерируемый объект.
Каждый `chunk` — маленький кусок ответа (несколько токенов/слов).
```python
for chunk in response:
    if chunk.choices[0].delta.content:
        print(chunk.choices[0].delta.content, end="", flush=True)
```

## Глава 13 — Агенты

**Q67. LLM всегда возвращает JSON?**

Нет! Только если явно попросить в системном промпте.
Даже тогда модель иногда добавляет пояснения вокруг JSON или делает ошибки.
Поэтому реальный код всегда оборачивает `json.loads` в `try/except`:
```python
try:
    result = json.loads(response.content)
except json.JSONDecodeError:
    # модель не вернула валидный JSON — обработать ошибку
    result = {"error": "invalid response", "raw": response.content}
```

**Q68. Как Python понимает что LLM «думает»?**

Никак. Python не «понимает» мышление.
ReAct — это паттерн промптинга: в системном промпте модели объяснено
«сначала напиши Thought (мысль), потом Action (действие), потом Observation (результат)».
Модель следует этому формату в тексте. Python парсит этот текст и извлекает JSON с `tool` и `args`.

**Q69. Функция как значение словаря?**

Да! Помните Q45 — функции это объекты. Их можно класть куда угодно:
```python
def check_fraud(claim): ...
def calculate(claim): ...

# Функции в словаре — реестр обработчиков
handlers = {
    "check_fraud": check_fraud,
    "calculate":   calculate,
}

# Динамический вызов
action = "check_fraud"
result = handlers[action](my_claim)   # вызвали нужную функцию по имени
```
Это паттерн «Strategy» или «Command» — очень распространён в агентах и плагинах.
